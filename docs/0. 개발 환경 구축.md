
# 0. 개발 환경 구축

---

## 0.1 왜 개발 환경 설정이 필요한가

프로그래밍을 배우려는 사람들이 가장 먼저 마주하는 난관은 코드 자체가 아니라 '환경'입니다. 온라인 강의를 따라 똑같이 타이핑했는데 "ModuleNotFoundError"가 뜨거나, 친구 컴퓨터에서는 잘 되던 코드가 내 컴퓨터에서는 실행조차 안 되는 상황이 발생합니다. 이런 경험이 쌓이면 "나는 프로그래밍에 재능이 없나?" 하는 착각에 빠지기 쉽습니다.

하지만 실상은 다릅니다. 프로그래밍 학습 초기에 겪는 문제의 약 90%는 '코드 작성 능력'이 아니라 '환경 설정' 문제입니다. Python이 어디 설치되어 있는지, 어떤 버전을 쓰는지, 필요한 라이브러리는 제대로 설치되었는지와 같은 기반이 흔들리면 아무리 좋은 코드를 작성해도 작동하지 않습니다.

반대로 환경을 한 번 제대로 설정해두면, 이후 모든 학습은 놀라울 정도로 순탄해집니다. 대부분의 에러 메시지가 사라지고, 오롯이 '어떻게 데이터를 다룰 것인가'에만 집중할 수 있습니다. 특히 데이터 분석과 인공지능 분야는 NumPy, Pandas, TensorFlow 같은 외부 라이브러리에 크게 의존하는데, 이들을 체계적으로 관리하지 않으면 버전 충돌로 인한 혼란이 끊이지 않습니다.

우리가 구축할 견고한 기반은 다음 네 가지 요소가 제 역할을 맡아 작동할 때 비로소 완성됩니다.

- **인터프리터 (Interpreter): 파이썬의 '심장'** 작성한 코드를 컴퓨터가 이해할 수 있는 언어로 한 줄씩 해석하고 실행하는 핵심 프로그램입니다. `python.exe` 혹은 `python3`라는 이름의 실행 파일 형태로 존재하며, 우리가 작성한 명령을 수행하는 실질적인 주체입니다.
    
- **패키지 (Packages): 기능을 확장하는 '부품'** 파이썬의 기본 기능에 날개를 달아주는 외부 도구들입니다. 데이터 분석을 위한 `pandas`, 시각화를 위한 `matplotlib`처럼 특정 목적에 특화된 부품들을 가져와 설치함으로써 파이썬의 가능성을 무한히 확장할 수 있습니다.
    
- **가상환경 (Virtual Environment): 독립된 '작업실'** 프로젝트마다 독립된 실행 공간을 만드는 기술입니다. 한 컴퓨터 내에서도 프로젝트 A는 구버전 라이브러리를, 프로젝트 B는 신버전을 써야 할 때가 있습니다. 가상환경은 이들이 서로 간섭하거나 충돌하지 않도록 방을 나누어 보호해 줍니다.
    
- **편집기 (Editor/IDE): 코드가 탄생하는 '창'** 개발자가 코드를 직접 입력하고 실행 명령을 내리는 인터페이스입니다. **VS Code**나 **Jupyter Notebook** 같은 도구가 여기에 해당하며, 문법 강조나 자동 완성 기능을 통해 개발의 효율성을 극대화합니다.

이 장에서는 앞으로의 모든 학습을 지탱할 견고한 기반을 만듭니다. Python 인터프리터, VS Code라는 코드 편집기, 가상환경, 그리고 패키지라는 네 가지 도구를 설치하고 연결하는 과정입니다.

## 0.2 Python 설치

### 0.2.1 Python 버전 선택

Python은 계속 진화하는 언어입니다. 2026년 2월 현재, Python 3.14가 정식 안정 버전으로 출시되었으나, 입문자를 위한 학습 환경으로는 **Python 3.13**을 권장합니다. 3.13은 파이썬의 성능을 비약적으로 높인 최신 기술이 반영되어 있으면서도, 현재 충분한 안정성을 확보했고, NumPy, Pandas, scikit-learn 등 주요 데이터 분석 라이브러리들도 대부분 3.13 지원을 완료한 상태입니다.

Python을 설치하는 방법은 크게 두 가지입니다. 공식 사이트(python.org)에서 직접 설치하거나, Anaconda라는 패키지를 설치하는 것입니다. Anaconda는 Python과 함께 데이터 분석용 라이브러리 수백 개를 한 번에 설치해주는 편리함이 있지만, 용량이 크고(약 3GB) 우리가 당장 필요하지 않은 도구까지 포함되어 있습니다.

이 책에서는 **공식 Python + pip** 방식을 권장합니다. 필요한 것만 설치하고, 가상환경을 통해 프로젝트별로 관리하는 방식이 실무에서 더 널리 쓰이며, 환경 관리의 원리를 이해하는 데도 유리합니다.

### 0.2.2 설치 과정

**Windows 사용자:**

1. 웹 브라우저에서 `python.org`에 접속합니다.
2. 상단 메뉴에서 Downloads → Windows를 선택합니다.
3. Python 3.13.x 버전의 설치 파일을 다운로드합니다. (x는 최신 패치 번호)
4. 다운로드한 파일을 실행하면 설치 창이 뜹니다.
5. **매우 중요:** 하단의 "Add Python 3.13 to PATH" 체크박스를 **반드시 선택**합니다. 이 한 가지를 놓치면 이후 모든 작업에서 Python을 찾지 못하는 문제가 발생합니다.
6. "Install Now"를 클릭하고 설치를 진행합니다.

**macOS 사용자:**

macOS는 기본적으로 Python 2.7이 설치되어 있지만, 이는 구버전이므로 사용하지 않습니다.

1. `python.org`에서 macOS용 Python 3.13.x 설치 파일을 다운로드합니다.
2. pkg 파일을 실행하고 안내에 따라 설치합니다.

**Linux 사용자:**

대부분의 배포판에 Python 3가 기본 설치되어 있지만, 버전 확인이 필요합니다.

```bash
python3 --version
```

만약 3.10 미만이라면 패키지 매니저로 업데이트합니다.

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.13

# Fedora
sudo dnf install python3.13
```

**설치 확인:**

모든 운영체제에서 공통적으로, 터미널(Windows는 PowerShell 또는 명령 프롬프트)을 열고 다음 명령어를 입력합니다.

```bash
python --version
```

또는

```bash
python3 --version
```

`Python 3.13.x` 같은 메시지가 출력되면 성공입니다. 만약 "python은 알 수 없는 명령입니다" 같은 오류가 나온다면 PATH 설정이 안 된 것이므로, Python을 다시 설치하되 "Add to PATH" 옵션을 체크해야 합니다.

## 0.3 VS Code 설치 및 설정

### 0.3.1 VS Code 다운로드 및 설치

Visual Studio Code(VS Code)는 Microsoft가 개발한 무료 코드 편집기입니다. 가볍고 빠르면서도 확장 기능을 통해 거의 모든 프로그래밍 언어를 지원합니다. Python 개발에서는 사실상 업계 표준 도구 중 하나입니다.

1. 웹 브라우저에서 `code.visualstudio.com`에 접속합니다.
2. 운영체제에 맞는 버전을 자동으로 감지해 다운로드 버튼을 보여줍니다.
3. 설치 파일을 실행합니다.
4. 설치 과정에서 다음 옵션들을 체크하는 것을 권장합니다:
    - "PATH에 추가" (Add to PATH)
    - "탐색기의 파일 상황에 맞는 메뉴에 '코드로 열기' 작업 추가" (Windows)
    - "탐색기의 디렉터리 상황에 맞는 메뉴에 '코드로 열기' 작업 추가" (Windows)

설치가 완료되면 VS Code를 실행합니다. 처음 열면 깔끔한 화면에 "Get Started" 탭이 보입니다.

### 0.3.2 필수 확장(Extension) 설치

VS Code의 진짜 힘은 확장 기능에서 나옵니다. 왼쪽 사이드바에서 네 개의 사각형이 겹쳐진 아이콘(Extensions)을 클릭합니다. 또는 단축키 `Ctrl+Shift+X` (Windows/Linux) 또는 `Cmd+Shift+X` (macOS)를 누릅니다.

검색창에 "Python"을 입력하면 여러 확장이 나오는데, **Microsoft에서 만든 "Python" 확장**을 설치합니다. 아이콘에 Python 로고가 있고 설치 횟수가 수천만 건에 달하는 것이 공식 확장입니다. "Install" 버튼을 클릭합니다.

이 확장이 제공하는 기능들:

- **자동완성(IntelliSense):** 코드를 타이핑하면 함수명, 변수명을 자동으로 제안합니다.
- **린팅(Linting):** 문법 오류나 잠재적 문제를 실시간으로 표시합니다.
- **디버깅:** 코드 실행을 중간에 멈추고 변수 값을 확인할 수 있습니다.
- **테스트 실행:** 나중에 pytest 같은 도구를 쓸 때 편리합니다.

설치가 완료되면 VS Code를 재시작할 필요는 없습니다. 바로 Python 파일을 만들면 확장이 활성화됩니다.

### 0.3.3 기본 설정

VS Code는 기본 상태로도 충분히 쓸 만하지만, 몇 가지 설정을 조정하면 더 편리합니다.

**자동 저장 활성화:**

코드를 작성하다 저장을 깜빡하는 경우가 많습니다. 자동 저장을 켜두면 이런 실수를 방지할 수 있습니다.

- 상단 메뉴: `File > Auto Save`를 클릭해 체크 표시가 나타나게 합니다.

**들여쓰기 설정:**

Python은 들여쓰기로 코드 블록을 구분하는 언어입니다. 탭과 스페이스를 섞어 쓰면 오류가 발생합니다. VS Code의 Python 확장은 자동으로 스페이스 4칸을 사용하도록 설정되어 있습니다. 확인 방법은 하단 상태바를 보면 "Spaces: 4"라고 표시됩니다.

**테마와 폰트 (선택사항):**

장시간 코드를 보려면 눈의 피로를 줄이는 것이 중요합니다.

- `File > Preferences > Color Theme` (Windows/Linux) 또는 `Code > Preferences > Color Theme` (macOS)
- 어두운 테마를 선호한다면 "Dark+ (default dark)" 또는 "Monokai"
- 밝은 테마를 선호한다면 "Light+ (default light)"

폰트 크기는 `File > Preferences > Settings`에서 "Font Size"를 검색해 조정할 수 있습니다. 기본값은 14인데, 모니터 크기에 따라 12~16 사이에서 선택하면 됩니다.

## 0.4 첫 Python 파일 실행

### 0.4.1 작업 폴더 생성 및 열기

프로그래밍 작업을 체계적으로 관리하려면 전용 폴더를 만드는 것이 좋습니다. 바탕화면이나 문서 폴더에 `python_basic`이라는 이름의 새 폴더를 만듭니다.

VS Code로 돌아와서:

1. `File > Open Folder`를 클릭합니다.
2. 방금 만든 `python_basic` 폴더를 선택하고 "폴더 선택" 버튼을 클릭합니다.

왼쪽 Explorer 패널에 `PYTHON_BASIC` 이름과 함께 폴더 구조가 나타납니다. 아직 파일이 없으므로 빈 상태입니다.

### 0.4.2 hello.py 작성 및 실행

폴더 이름 옆의 "새 파일" 아이콘을 클릭하거나, `File > New File > Python File`를 클릭합니다. 파일 이름을 `hello.py`로 입력합니다. `.py` 확장자는 Python 파일을 의미합니다.

파일이 열리면 다음 코드를 입력합니다.

```python
print("Hello, Python!")
```

`print`는 화면에 메시지를 출력하는 함수입니다. 괄호 안의 큰따옴표(또는 작은따옴표) 사이에 쓴 텍스트가 그대로 출력됩니다.

파일을 저장합니다: `Ctrl+S` (Windows/Linux) 또는 `Cmd+S` (macOS). 탭 이름 옆에 점(●)이 있었다면 저장 후 사라집니다.

**실행 방법 1: 재생 버튼 사용**

코드 편집 영역 우측 상단에 삼각형 재생 버튼(▶)이 보입니다. 이것을 클릭하면 하단에 터미널 패널이 열리면서 다음과 같은 출력이 나타납니다.

```
Hello, Python!
```


**실행 방법 2: 터미널에서 직접 실행**

하단의 터미널 패널에서 직접 명령어를 입력할 수도 있습니다. 터미널이 보이지 않으면 상단 메뉴에서 `View > Terminal`을 선택하거나 `Terminal > New Terminal`을 클릭합니다.

터미널에서 다음 명령어를 입력합니다.

```bash
python hello.py
```

macOS나 Linux에서 `python` 명령이 Python 2를 가리키는 경우가 있으므로, `python3 hello.py`를 사용해야 할 수도 있습니다.

`Hello, Python!`으로 출력 되면 성공입니다.

### 0.4.3 첫 에러 경험하기

에러 메시지는 무섭지 않습니다. 오히려 "무엇이 잘못됐는지" 알려주는 친절한 안내판입니다. 의도적으로 오류를 만들어 봅시다.

`hello.py` 파일을 다음과 같이 수정합니다.

```python
prin("Hello, Python!")
```

`print`를 `prin`으로 잘못 입력했습니다. 저장하고 실행하면 다음과 같은 메시지가 나타납니다.

```
Traceback (most recent call last):
  File "hello.py", line 1, in <module>
    prin("Hello, Python!")
NameError: name 'prin' is not defined. Did you mean: 'print'?
```

이 메시지를 읽는 법:

- `File "hello.py", line 1`: 문제가 발생한 파일과 줄 번호
- `prin("Hello, Python!")`: 문제가 된 코드
- `NameError: name 'prin' is not defined`: `prin`이라는 이름이 정의되지 않았다는 뜻. Python은 `print`는 알지만 `prin`은 모릅니다.

에러를 고치려면 `prin`을 `print`로 되돌립니다. 이렇게 에러 메시지를 읽고 수정하는 과정이 프로그래밍의 일상입니다.

## 0.5 가상환경 (Virtual Environment)

### 0.5.1 가상환경이 필요한 이유

상황을 하나 가정해봅시다. 당신은 두 개의 데이터 분석 프로젝트를 진행 중입니다.

- **프로젝트 A**: 1년 전 시작했고, NumPy 1.26 버전을 사용합니다.
- **프로젝트 B**: 이번 주에 시작했고, 최신 NumPy 2.4 버전이 필요합니다.

만약 컴퓨터에 Python과 라이브러리를 전역으로(globally) 설치했다면, NumPy는 하나의 버전만 설치할 수 있습니다. 프로젝트 B를 위해 NumPy를 2.4로 업데이트하면, 프로젝트 A의 코드가 작동하지 않을 수 있습니다. 반대로 1.26을 유지하면 프로젝트 B에서 새 기능을 쓸 수 없습니다.

가상환경은 이 문제를 해결합니다. 각 프로젝트마다 독립된 Python 환경을 만들어, 마치 각자의 서랍을 가진 것처럼 라이브러리를 따로 관리합니다. 프로젝트 A의 서랍에는 NumPy 1.26이, 프로젝트 B의 서랍에는 NumPy 2.4가 들어 있고, 둘은 서로 간섭하지 않습니다.

실무에서는 가상환경 사용이 필수입니다. 협업할 때 팀원들과 동일한 환경을 공유하고, 배포할 때 정확히 어떤 라이브러리 버전을 써야 하는지 명시할 수 있습니다.

### 0.5.2 venv로 가상환경 생성

Python은 3.3 버전부터 `venv`라는 가상환경 도구를 기본으로 제공합니다. 별도 설치 없이 바로 사용할 수 있습니다.

VS Code 하단의 터미널에서 다음 명령어를 입력합니다. (현재 작업 폴더가 `python_basic`인지 확인하세요)

```bash
python -m venv venv
```

명령어 해석:

- `python -m venv`: Python의 venv 모듈을 실행합니다.
- 마지막 `venv`: 만들 가상환경 폴더의 이름입니다. 보통 `venv`, `env`, `.venv` 같은 관례적 이름을 사용합니다.

명령어를 실행하면 몇 초간 아무 반응이 없다가, 다시 프롬프트가 나타납니다. 왼쪽 Explorer 패널을 보면 `venv`라는 새 폴더가 생성되어 있습니다.

이 폴더 안에는 무엇이 들어 있을까요? 클릭해서 펼쳐보면 `Scripts` (Windows) 또는 `bin` (macOS/Linux) 폴더, `Lib` 폴더 등이 보입니다. 여기에 독립된 Python 인터프리터와 pip, 그리고 설치할 라이브러리들이 저장됩니다.

### 0.5.3 가상환경 활성화

가상환경을 만들었다고 자동으로 사용되는 것은 아닙니다. 명시적으로 "이 가상환경을 사용하겠다"고 활성화해야 합니다.

**Windows (명령 프롬프트):**

```bash
venv\Scripts\activate.bat
```

**Windows (PowerShell):**

```bash
venv\Scripts\Activate.ps1
```

만약 다음과 같은 에러가 발생한다면:

```
... cannot be loaded because running scripts is disabled on this system.
```

이것은 PowerShell의 보안 정책 때문입니다. 다음 명령어로 정책을 변경합니다.

```bash
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

"Y"를 입력해 확인하고, 다시 활성화 명령어를 실행합니다.

**macOS / Linux:**

```bash
source venv/bin/activate
```

활성화에 성공하면 터미널 프롬프트 앞에 `(venv)`가 표시됩니다.

```
(venv) C:\Users\YourName\python_basic>
```

이 상태가 되면 이제 Python 명령어와 pip는 전역이 아닌 이 가상환경 내의 것을 사용합니다.

### 0.5.4 VS Code에서 인터프리터 선택

VS Code는 여러 Python 인터프리터 중 어떤 것을 사용할지 선택할 수 있습니다. 가상환경을 만들었다면 이것을 VS Code에 알려줘야 합니다.

1. 키보드에서 `Ctrl+Shift+P` (Windows/Linux) 또는 `Cmd+Shift+P` (macOS)를 누릅니다. 명령 팔레트가 열립니다.
2. "Python: Select Interpreter"를 입력하거나 목록에서 선택합니다.
3. 여러 Python 경로가 나타나는데, 그중에서 `./venv/Scripts/python.exe` (Windows) 또는 `./venv/bin/python` (macOS/Linux)를 선택합니다.

선택하면 VS Code 하단 상태바 오른쪽에 Python 버전과 함께 `('venv': venv)` 같은 표시가 나타납니다. 이제 VS Code에서 실행하는 모든 Python 코드는 이 가상환경을 사용합니다.

### 0.5.5 가상환경 비활성화

작업이 끝나고 가상환경에서 나오려면 터미널에서 다음 명령어를 입력합니다.

```bash
deactivate
```

프롬프트 앞의 `(venv)` 표시가 사라집니다. 다시 가상환경을 쓰려면 활성화 명령어를 실행하면 됩니다.

## 0.6 라이브러리 설치 및 관리

### 0.6.1 pip 기본 사용법

`pip`는 Python 패키지를 설치하고 관리하는 도구입니다. 2014년에 출시된 Python 3.4 이상을 설치하면 자동으로 함께 설치됩니다.

가상환경이 활성화된 상태에서 터미널에 다음 명령어를 입력합니다.

```bash
pip install numpy
```

몇 초 후 다음과 같은 메시지가 나타납니다.

```
Collecting numpy
  Downloading numpy-2.4.x-...whl (xx.x MB)
Installing collected packages: numpy
Successfully installed numpy-2.4.x
```

NumPy가 설치되었습니다. 여러 라이브러리를 한 번에 설치하려면 공백으로 구분합니다.

```bash
pip install pandas matplotlib
```

설치가 제대로 됐는지 확인해봅시다. `hello.py` 파일을 열고 내용을 다음과 같이 수정합니다.

```python
import numpy as np
print(f"NumPy 버전: {np.__version__}")
```

실행하면 다음과 같이 출력됩니다.

```
NumPy 버전: 2.4.x
```

`import` 키워드는 라이브러리를 불러오는 명령입니다. `as np`는 긴 이름을 짧게 줄여 쓰겠다는 뜻입니다. `np.__version__`은 설치된 NumPy의 버전 정보를 담고 있는 속성입니다.

### 0.6.2 requirements.txt 작성

프로젝트가 커지면 사용하는 라이브러리가 수십 개가 됩니다. 이것을 일일이 기억하거나 수동으로 다시 설치하기는 어렵습니다. `requirements.txt`는 프로젝트에 필요한 모든 라이브러리와 버전을 기록하는 파일입니다.

현재 가상환경에 설치된 모든 라이브러리 목록을 파일로 저장하려면:

```bash
pip freeze > requirements.txt
```

`python_basic` 폴더에 `requirements.txt` 파일이 생성됩니다. 열어보면 다음과 같은 내용이 있습니다.

```
numpy==2.4.2
pandas==3.0.1
matplotlib==3.10.8
...
```

각 줄은 `라이브러리이름==버전` 형식입니다.

이 파일의 진가는 다른 환경에서 드러납니다. 예를 들어 새 컴퓨터로 옮기거나, 팀원과 협업하거나, 몇 달 뒤 프로젝트를 다시 시작할 때, 다음 명령어 하나로 동일한 환경을 재현할 수 있습니다.

```bash
pip install -r requirements.txt
```

`-r` 옵션은 "파일을 읽어서(read) 그 안의 모든 패키지를 설치하라"는 뜻입니다.

### 0.6.3 자주 쓰는 pip 명령어

**설치된 라이브러리 목록 보기:**

```bash
pip list
```

패키지 이름과 버전이 표 형태로 출력됩니다.

**특정 라이브러리 삭제:**

```bash
pip uninstall numpy
```

확인 메시지가 나타나면 "Y"를 입력합니다.

**라이브러리 업데이트:**

```bash
pip install --upgrade numpy
```

최신 버전이 있으면 다운로드하고 설치합니다.

**특정 버전 설치:**

```bash
pip install numpy==1.24.0
```

등호 두 개(\=\=)로 정확한 버전을 지정합니다.

## 0.7 VS Code 디버거 맛보기

### 0.7.1 Breakpoint 설정

디버거는 코드 실행을 중간에 멈추고, 그 순간의 변수 값을 확인할 수 있는 강력한 도구입니다. 간단한 예제로 사용법을 익혀봅시다.

`debug_test.py` 파일을 새로 만들고 다음 코드를 입력합니다.

```python
numbers = [1, 2, 3, 4, 5]
total = 0
for num in numbers:
    total = total + num
print(f"합계: {total}")
```

3번 줄(`total = total + num`)의 줄 번호 왼쪽 여백을 클릭하면 빨간 점이 생깁니다. 이것이 Breakpoint(중단점)입니다.

키보드에서 `F5`를 누르면 디버그 모드로 실행됩니다. 처음 실행 시 "디버그 구성 선택" 메뉴가 나타나면 "Python File"을 선택합니다.

코드가 실행되다가 빨간 점이 있는 줄에서 멈춥니다. 왼쪽 패널에 "Variables" 섹션이 나타나고, 현재 정의된 변수들과 그 값이 보입니다.

- `numbers`: [1, 2, 3, 4, 5]
- `total`: 0
- `num`: 1

상단의 디버그 컨트롤 바에서 "Step Over" 버튼(또는 `F10`)을 누르면 한 줄씩 실행됩니다. `total` 값이 0에서 1로, 1에서 3으로 변하는 것을 실시간으로 관찰할 수 있습니다.

### 0.7.2 언제 사용하는가

디버거는 다음 상황에서 유용합니다.

- "왜 이 변수에 이 값이 들어 있지?" 하는 의문이 들 때
- 반복문 안에서 값이 어떻게 변하는지 추적할 때
- 복잡한 조건문에서 어느 경로로 실행되는지 확인할 때

지금은 간단한 예제라 `print`로도 충분하지만, 3장(제어문)부터는 디버거 없이는 이해하기 어려운 상황이 자주 등장합니다. 그때 다시 사용법을 안내하겠습니다.

## 0.8 문제 해결 체크리스트

코드를 작성하다 보면 "분명 맞는데 왜 안 되지?" 하는 순간이 옵니다. 당황하지 말고 다음 항목을 순서대로 확인하세요.

**1. 가상환경이 활성화되어 있나요?**

터미널 프롬프트 앞에 `(venv)` 표시가 있는지 확인합니다. 없다면 `venv\Scripts\Activate.ps1` (Windows) 또는 `source venv/bin/activate` (macOS/Linux)를 실행합니다.

**2. VS Code가 올바른 인터프리터를 선택했나요?**

하단 상태바 오른쪽을 보면 Python 버전과 환경이 표시됩니다. `('venv': venv)` 같은 표시가 있어야 합니다. 없다면 `Ctrl+Shift+P` → "Python: Select Interpreter"로 가상환경을 선택합니다.

**3. 필요한 라이브러리가 설치되어 있나요?**

터미널에서 `pip list`를 실행해 확인합니다. 없다면 `pip install 라이브러리이름`으로 설치합니다.

**4. 파일을 저장했나요?**

파일 탭 이름 옆에 점(●)이 있으면 저장되지 않은 상태입니다. `Ctrl+S` 또는 `Cmd+S`로 저장합니다.

**5. 들여쓰기 오류가 없나요?**

Python은 탭과 스페이스를 구분합니다. 한 파일 안에서 둘을 섞어 쓰면 `IndentationError`가 발생합니다. VS Code는 기본적으로 스페이스를 사용하므로, 탭 키를 눌러도 자동으로 스페이스 4개로 변환됩니다.

**6. 에러 메시지를 읽었나요?**

에러 메시지는 무섭지 않습니다. 마지막 줄의 `NameError`, `TypeError` 같은 단어와 그 뒤의 설명을 주의 깊게 읽으면 대부분의 문제는 해결 힌트가 있습니다.

## 0.9 실습

환경 설정이 제대로 되었는지 최종 점검하는 스크립트를 작성해봅시다. `check_env.py` 파일을 만들고 다음 코드를 입력합니다.

```python
import sys

print("=" * 50)
print("Python 환경 점검")
print("=" * 50)

print(f"Python 버전: {sys.version}")
print(f"실행 경로: {sys.executable}")

print("\n라이브러리 확인:")

try:
    import numpy as np
    print(f"  NumPy: {np.__version__} ✓")
except ImportError:
    print("  NumPy: 미설치 - pip install numpy 실행 필요")

try:
    import pandas as pd
    print(f"  Pandas: {pd.__version__} ✓")
except ImportError:
    print("  Pandas: 미설치 - pip install pandas 실행 필요")

try:
    import matplotlib
    print(f"  Matplotlib: {matplotlib.__version__} ✓")
except ImportError:
    print("  Matplotlib: 미설치 - pip install matplotlib 실행 필요")

print("=" * 50)
print("환경 점검 완료!")
print("=" * 50)
```

저장하고 실행합니다. 성공적으로 설정되었다면 다음과 같은 출력이 나타납니다.

```
==================================================
Python 환경 점검
==================================================
Python 버전: 3.13.x ...
Python 실행 경로: C:\Users\YourName\python_basic\venv\Scripts\python.exe

라이브러리 확인:
  NumPy: 2.4.2 ✓
  Pandas: 3.0.1 ✓
  Matplotlib: 3.10.8 ✓
==================================================
환경 점검 완료!
==================================================
```

**핵심 확인 사항:**

1. 실행 경로에 `venv` 폴더가 포함되어 있는가? (가상환경 사용 중)
2. 세 라이브러리 모두 버전과 함께 ✓ 표시가 나타나는가?

두 조건이 모두 충족되면 환경 설정이 완벽하게 완료된 것입니다. 이제 1장으로 넘어가 Python 문법을 배울 준비가 되었습니다.
